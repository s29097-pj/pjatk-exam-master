💡 Aha! Moment – Frontend to „Budynek, w którym użytkownik pije kawę na tarasie”:

Wyobraź sobie, że wchodzisz do restauracji. HTML to szkielet budynku – ściany nośne, okna, drzwi i układ pomieszczeń. Bez niego wszystko by się zawaliło, ale gołe cegły nie są zbyt przyjemne. CSS to architektura wnętrz: kolory ścian, fonty, zasłony, rozmieszczenie mebli. Sprawia, że gość chce zostać dłużej. JavaScript to kelner, który słucha Twoich zamówień (kliknięć, przeciągnięć) i dynamicznie przynosi dania (dane z serwera) bez przeładowywania całego budynku. Frameworki (React, Angular, Vue) to gotowe, prefabrykowane moduły – zamiast samemu mieszać beton i układać cegły, Ty składasz ściany z gotowych paneli, skupiając się na wyjątkowym charakterze lokalu.

🏷️ Kryteria oceny i Słowa kluczowe (Active Recall): HTML5 – semantyka (header, nav, main, article) CSS3 – Flexbox, Grid, Responsywność (RWD) JavaScript (ES6+) – DOM, fetch/axios Frameworki: React (JSX, hooks), Angular, Vue (SFC) Stan aplikacji (State Management) SPA (Single Page Application) Web Components, Shadow DOM TypeScript (typowanie w frontendzie) Webpack, Vite (bundlery) i npm (menedżer pakietów) Dostępność (Accessibility, a11y) – ARIA, kontrast Web APIs (geolocation, canvas, lokalne magazyny)
1. HTML5 – Szkielet i semantyka dokumentu

HTML (HyperText Markup Language) nie jest językiem programowania – to język znaczników opisujący strukturę dokumentu. HTML5 wprowadził semantyczne znaczniki (<header>, <nav>, <main>, <article>, <footer>), które mówią przeglądarce i robotom indeksującym (SEO), jaką rolę pełni dany blok. Formularze w HTML5 oferują natywną walidację (required, pattern, typy pól jak email/date). Canvas i SVG umożliwiają rysowanie grafiki bezpośrednio w przeglądarce.

🗣️ Jak to powiedzieć: „Proszę Państwa, HTML to nie język programowania – to język znaczników, szkielet każdej strony. HTML5 wprowadził semantyczne znaczniki: header, nav, main, article, footer – one mówią przeglądarce i wyszukiwarkom, co jest czym. Do tego formularze z natywną walidacją, canvas i SVG do grafiki. To fundament, na którym stoi CSS i JavaScript. Proszę spojrzeć na planszę – pokazuje ona układ semantycznych bloków na stronie.”
2. CSS3 – Stylizacja, układ i responsywność

CSS (Cascading Style Sheets) oddziela warstwę prezentacji od struktury. Współczesne techniki układu (Flexbox, CSS Grid) pozwalają na tworzenie złożonych, elastycznych siatek bez używania tabel czy floatów. Responsive Web Design (RWD) z Media Queries dostosowuje wygląd do szerokości ekranu (mobile-first). Preprocesory jak Sass/SCSS dodają zmienne, zagnieżdżanie i mixiny, ułatwiając zarządzanie dużymi arkuszami stylów.

🗣️ Jak to powiedzieć: „CSS3 to warstwa prezentacji. Mamy dziś dwa potężne narzędzia: Flexbox – jednowymiarowy, świetny do pasków nawigacji i centrowania. CSS Grid – dwuwymiarowy, do budowy całych layoutów strony. Do tego Media Queries dają responsywność – strona dostosowuje się do ekranu telefonu, tabletu i monitora. Preprocesory jak Sass dodają zmienne i zagnieżdżanie, ułatwiając pracę z dużymi projektami. Proszę spojrzeć na planszę – porównuje ona Flexbox, Grid i Media Queries.”
3. JavaScript (ES6+) – Język interakcji i asynchroniczności

JavaScript to jedyny natywny język programowania przeglądarek. Operuje na DOM (Document Object Model) – dynamicznie modyfikuje strukturę HTML i CSS. Współczesny JS (ES6+) oferuje let/const, funkcje strzałkowe, destrukturyzację, moduły (import/export) i programowanie asynchroniczne (async/await, fetch). Silniki JS (V8 w Chrome, SpiderMonkey w Firefox) kompilują kod w locie (JIT), osiągając wydajność zbliżoną do języków kompilowanych.

🗣️ Jak to powiedzieć: „JavaScript to jedyny język programowania działający natywnie w przeglądarce. Operuje na DOM – może dynamicznie zmieniać HTML i CSS. Współczesny JS, od ES6 wzwyż, oferuje async/await do asynchronicznego pobierania danych z API, funkcje strzałkowe, moduły. Silniki jak V8 kompilują go w locie, więc jest bardzo szybki. To warstwa logiki i interakcji. Proszę spojrzeć na planszę – pokazuje ona komunikację między przeglądarką, JS i serwerem API.”
4. Frameworki i biblioteki – „Gotowe prefabrykaty budowlane”
🗣️ Jak to powiedzieć: „Frameworki frontendowe to gotowe prefabrykaty. React od Mety to biblioteka do budowy interfejsów – komponenty, JSX, Virtual DOM. Angular od Google to pełny framework w TypeScripcie z modułami i wstrzykiwaniem zależności. Vue.js to progresywny framework – lekki, reaktywny, z Composition API. Każdy ma swoje zalety, ale łączy je jedno: pozwalają budować złożone aplikacje SPA bez przeładowywania strony. Proszę spojrzeć na planszę – porównuje ona te trzy frameworki.”
⚛️ React (Meta)
Biblioteka UI. JSX, komponenty funkcyjne, hooks (useState, useEffect). Virtual DOM. Zarządzanie stanem: Redux / Zustand.
🅰️ Angular (Google)
Pełny framework MVC. TypeScript, moduły, serwisy, dependency injection. Dwukierunkowe wiązanie danych ([(ngModel)]).
🟢 Vue.js (E. You)
Progresywny framework. Single File Components (.vue). Reaktywność, Composition API. Łagodna krzywa uczenia się.
⚠️ Pułapki egzaminacyjne:
  • HTML to nie język programowania! To język znaczników. Nie ma w nim instrukcji warunkowych ani pętli – do tego służy JavaScript i frameworki.
  • CSS Grid ≠ Flexbox. Grid jest dwuwymiarowy (wiersze i kolumny jednocześnie), Flexbox jest jednowymiarowy (wiersz albo kolumna). Nie zastępują się nawzajem – uzupełniają się.
  • React to biblioteka, Angular to framework. React zajmuje się tylko widokiem (V w MVC), podczas gdy Angular dostarcza kompletny ekosystem (routing, HTTP, DI).
🗣️ Jak odpowiedzieć komisji na pytanie o pułapki: „Komisjo, trzy kluczowe pułapki. Po pierwsze – HTML to język znaczników, nie programowania. Po drugie – CSS Grid i Flexbox się nie zastępują: Grid jest dwuwymiarowy, Flexbox jednowymiarowy. Po trzecie – React to biblioteka do widoków, Angular to pełny framework. Warto też pamiętać, że nowoczesny frontend to nie tylko te trzy technologie, ale też narzędzia: Webpack, Vite, TypeScript i menedżery stanu jak Redux.”
